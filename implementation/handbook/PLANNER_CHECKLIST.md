# Planner Checklist

主审模型为每个模块出计划时必须逐项确认：

- [ ] 仓库当前分支和起始 commit 已记录。
- [ ] 前置模块已由主审模型标记 `APPROVED`，不是只有本地模型自称完成。
- [ ] 已阅读所有将被修改目录中的现有实现和约束。
- [ ] 模块目标可以由可观察结果验证，不使用“基本完成”“支持”等模糊表述。
- [ ] 允许路径最小化，禁止路径明确。
- [ ] 产品目录结构和各文件职责已预先列出。
- [ ] 数据结构、接口、错误行为、事务边界和幂等要求明确。
- [ ] 生成步骤适合 6B 模型：小步、顺序固定、每步有局部测试。
- [ ] 单元、集成、迁移、回归和静态检查命令明确。
- [ ] Manifest 和 Local Report 字段足够支持后续独立审查。
- [ ] 回滚点、禁止行为和停止条件明确。
- [ ] 本规划窗口未代替本地模型编写产品代码。
