# Reviewer Checklist

本地模型提交后，主审模型必须独立检查：

1. **提交边界**：diff 是否只涉及允许路径；是否混入格式化、依赖升级或后续模块。
2. **Manifest 一致性**：每个 changed file 是否都在 Manifest；职责描述是否与代码一致。
3. **结构标准**：路由、领域逻辑、持久化、迁移、测试是否分层；是否把所有逻辑堆进单文件。
4. **真实性**：是否存在 TODO、假数据、内存替代持久化、永远返回成功、硬编码测试结果。
5. **数据安全**：事务、外键、唯一约束、租户字段、删除语义、时间字段是否正确。
6. **错误处理**：是否吞异常、返回模糊错误、把可重试错误当成功。
7. **测试质量**：是否覆盖失败路径、边界、隔离、幂等；是否只验证 mock 被调用。
8. **独立回测**：重新运行规划命令并记录真实输出。
9. **兼容性**：现有 API 和测试是否回归通过。
10. **判定**：只有无硬失败且达到模块门槛才能 `APPROVED`。
